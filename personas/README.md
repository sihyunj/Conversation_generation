# US English Messaging Personas

Synthetic English-language personas tailored for **messaging / chat conversation data generation**.
Inspired by [NVIDIA Nemotron-Personas-Korea](https://huggingface.co/datasets/nvidia/Nemotron-Personas-Korea),
but scoped to (1) US demographics and (2) messaging-specific traits (texting style, emoji use,
preferred platforms, slang register, etc.).

## Design principles

1. **US-grounded distributions**. Demographics (age, sex, region, ethnicity, education,
   marital status, occupation) approximately follow US Census adult population ratios.
   This is a *simplified* approach (no PGM); see `prompts/attributes_pool.json` for the
   exact distribution weights used.
2. **Messaging-first**. Beyond Nemotron's general persona fields we add 8 messaging-specific
   fields (`communication_style`, `emoji_density`, `abbreviation_use`, etc.) that drive
   downstream conversation-generation prompts.
3. **Diverse but realistic**. Random combinations are filtered through plausibility checks
   (e.g., a 78yo retiree is unlikely to use heavy Gen-Z slang on Discord).
4. **Topic combinations are filtered**. When pairing personas for conversation generation,
   we exclude pairs whose age gap, social distance, or platform overlap make the relationship
   implausible.

## Schema (summary — see `schema.md` for details)

- **Identity**: `uuid`, `first_name`, `last_name`, `sex`, `age`
- **Geography**: `region`, `state`, `city`
- **Background**: `ethnicity`, `education_level`, `marital_status`, `household_size`,
  `occupation`, `income_bracket`
- **Personality (Big Five 0–100)**: `openness`, `conscientiousness`, `extraversion`,
  `agreeableness`, `neuroticism`
- **Messaging traits**: `communication_style`, `emoji_density`, `abbreviation_use`,
  `avg_message_length`, `punctuation_style`, `capitalization_style`,
  `preferred_platforms` (list), `slang_register`
- **Narratives**: `professional_persona`, `social_persona`, `messaging_persona`,
  `hobbies_and_interests`, `core_values`

## Usage

### Read personas

```python
import json
with open("data/personas.jsonl") as f:
    personas = [json.loads(line) for line in f]
print(len(personas))  # 100
```

### Drop a persona into a system prompt

```python
p = personas[0]
system = f"""You are simulating {p['first_name']} {p['last_name']}, a {p['age']}yo {p['occupation']} from {p['city']}, {p['state']}.

Messaging style: {p['messaging_persona']}
Communication: {p['communication_style']}, emoji {p['emoji_density']}, slang {p['slang_register']}.

Stay fully in character when texting."""
```

### Use topic pairs for conversation generation

```python
with open("data/topics.jsonl") as f:
    topic_pairs = [json.loads(line) for line in f]

pair = topic_pairs[0]
# pair contains: persona_a_uuid, persona_b_uuid, relationship,
# platform, topic, opener_hint
```

## Regenerating

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=...

# generate 100 personas (or pass --count N)
python generate_personas.py --count 100 --out data/personas.jsonl

# generate topic pairs
python generate_topics.py --in data/personas.jsonl --out data/topics.jsonl --pairs 300

# validate
python validate.py
```

The shipped `data/personas.jsonl` and `data/topics.jsonl` were generated by Claude
(this assistant) directly — re-running the scripts with your own API key produces a
fresh batch with the same schema.

## Limitations

- **Simplified sampling**: Distributions are hand-tuned weights, not a true PGM grounded
  in raw census microdata. Joint distributions (e.g., occupation × education × region)
  are approximated, not exact.
- **Sex only**: Following Nemotron's convention, only biological sex (Male/Female) is
  modeled — gender identity and sexual orientation are not represented.
- **English-only, US-only**: All narratives are American English. Spanish-dominant
  bilinguals are noted via `native_language` but narratives stay English.
- **100 is small**: Enough to seed conversation generation, not enough to be a final
  fine-tuning corpus on its own. Scale up with the script.
